# Skill: docker-build

## Overview

**Skill ID**: `docker-build`
**Used by**: DockerAgent
**Purpose**: Generates Docker image build and tag instructions for the LightCI Server.

This skill defines the step-by-step procedure DockerAgent follows when generating
Docker build and tag commands. It covers prerequisite validation, Dockerfile review,
image build, tagging strategy, and local verification.

---

## Prerequisites

| Requirement | Minimum Version | Verification Command |
|---|---|---|
| Docker Engine or Desktop | 24+ | `docker --version` |
| Docker daemon running | — | `docker info` |
| `Dockerfile` | (project root) | `test -f Dockerfile` |
| JAR artifact | `target/light-ci-server.jar` | `test -f target/light-ci-server.jar` |
| Free disk space | 2 GB (for layers) | `docker system df` |

| Build Arg | Default | Description |
|---|---|---|
| `JAR_FILE` | `target/light-ci-server.jar` | Path to the fat JAR |
| `APP_PORT` | `8080` | Application listen port |
| `BASE_IMAGE` | `eclipse-temurin:17-jdk-alpine` | Builder stage base image |
| `RUNTIME_IMAGE` | `eclipse-temurin:17-jre-alpine` | Runtime stage base image |

## Step-by-Step Procedure

### Phase 1: Pre-Build Validation

```bash
# 1.1 — Check Docker daemon is running
docker info > /dev/null 2>&1 || { echo "ERROR: Docker not running"; exit 1; }

# 1.2 — Check Dockerfile exists
test -f Dockerfile || { echo "ERROR: Dockerfile not found"; exit 1; }

# 1.3 — Check JAR exists
test -f target/light-ci-server.jar || {
  echo "ERROR: JAR not found. Run Maven build first."
  exit 1
}

# 1.4 — Check available disk space (need ~2GB for layers)
docker system df
```

### Phase 2: Determine Image Reference

```bash
# 2.1 — Construct full image reference
IMAGE_REF="${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"
IMAGE_LATEST="${DOCKER_USERNAME}/${DOCKER_IMAGE}:latest"
echo "Building: ${IMAGE_REF}"
```

### Phase 3: Build the Image

```bash
# 3.1 — Build with platform targeting and build args
docker build \
  --platform linux/amd64 \
  --build-arg JAR_FILE=target/light-ci-server.jar \
  --build-arg APP_PORT=8080 \
  -t "${IMAGE_REF}" \
  -f Dockerfile \
  .

# Verify build succeeded
docker image inspect "${IMAGE_REF}" > /dev/null || {
  echo "ERROR: Image build failed"
  exit 1
}
echo "Built: $(docker image inspect ${IMAGE_REF} --format '{{.Id}}' | cut -c1-19)"
```

### Phase 4: Tag the Image

```bash
# 4.1 — Add 'latest' tag for convenience
docker tag "${IMAGE_REF}" "${IMAGE_LATEST}"
echo "Tagged: ${IMAGE_LATEST}"

# 4.2 — Optionally tag with git commit SHA for traceability
GIT_SHA=$(git rev-parse --short HEAD 2>/dev/null || echo "nogit")
docker tag "${IMAGE_REF}" "${DOCKER_USERNAME}/${DOCKER_IMAGE}:${GIT_SHA}"
echo "Tagged: ${DOCKER_USERNAME}/${DOCKER_IMAGE}:${GIT_SHA}"
```

### Phase 5: Verify Image Locally

```bash
# 5.1 — List the image
docker images "${DOCKER_USERNAME}/${DOCKER_IMAGE}"

# 5.2 — Inspect image size and metadata
docker image inspect "${IMAGE_REF}" \
  --format='Size: {{.Size}} bytes, Created: {{.Created}}'

# 5.3 — Optional: Run locally to confirm it starts
docker run -d --name lightci-verify -p 8080:8080 "${IMAGE_REF}"
sleep 20
HTTP_STATUS=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/actuator/health || echo "000")
echo "Health check: HTTP ${HTTP_STATUS}"
docker stop lightci-verify && docker rm lightci-verify
```

## Complete Generated Script

```bash
#!/bin/bash
# Generated by docker-build skill — LightCI Server
set -euo pipefail

DOCKER_USERNAME="${DOCKER_USERNAME:?Required}"
DOCKER_IMAGE="${DOCKER_IMAGE:-light-ci-server}"
DOCKER_TAG="${DOCKER_TAG:-latest}"
APP_PORT="${APP_PORT:-8080}"
IMAGE_REF="${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"

echo "Building: ${IMAGE_REF}"

# Validate
docker info > /dev/null 2>&1 || { echo "ERROR: Docker daemon not running"; exit 1; }
test -f Dockerfile       || { echo "ERROR: Dockerfile not found"; exit 1; }
test -f target/light-ci-server.jar || { echo "ERROR: JAR not found — run mvn first"; exit 1; }

# Build
docker build \
  --platform linux/amd64 \
  --build-arg JAR_FILE=target/light-ci-server.jar \
  --build-arg APP_PORT="${APP_PORT}" \
  -t "${IMAGE_REF}" \
  -f Dockerfile \
  .

# Tag
docker tag "${IMAGE_REF}" "${DOCKER_USERNAME}/${DOCKER_IMAGE}:latest"

# Report
echo "Build complete:"
docker images "${DOCKER_USERNAME}/${DOCKER_IMAGE}"
```

## How to Verify Success

```bash
# Image exists
docker images myusername/light-ci-server

# Image has non-zero size
docker image inspect myusername/light-ci-server:1.0.0 \
  --format='{{.Size}}' | awk '{printf "%.0f MB\n", $1/1048576}'

# Image contains expected layers (Spring Boot layered JAR)
docker history myusername/light-ci-server:1.0.0

# Image runs and health check passes
docker run --rm -p 8080:8080 myusername/light-ci-server:1.0.0 &
sleep 30
curl -f http://localhost:8080/actuator/health && echo "HEALTHY"
docker stop $(docker ps -q --filter ancestor=myusername/light-ci-server:1.0.0) 2>/dev/null || true
```

## Common Errors and Fixes

### `Cannot connect to the Docker daemon`
**Fix**: Start Docker Desktop (Mac/Windows) or `sudo systemctl start docker` (Linux).

### `COPY target/light-ci-server.jar: not found`
**Cause**: JAR was not built before docker build.
**Fix**: Run `mvn clean package -DskipTests` first.

### `error: failed to solve: eclipse-temurin:17-jre-alpine: not found`
**Cause**: Wrong base image name or Docker Hub rate limit.
**Fix**: Try `eclipse-temurin:17-jre` (without `-alpine`) or authenticate with Docker Hub.

### `no space left on device`
**Cause**: Docker image layers have filled the disk.
**Fix**: `docker system prune -f` to remove unused images and build cache.

### Platform warning: `image platform (linux/arm64) does not match the expected platform (linux/amd64)`
**Cause**: Building on Apple Silicon without specifying `--platform`.
**Fix**: Always include `--platform linux/amd64` for EC2 deployment.

### `HEALTHCHECK` failing during local test
**Cause**: Spring Boot hasn't fully started within the healthcheck interval.
**Fix**: Increase `--start-period` in the HEALTHCHECK instruction, or increase the `sleep` in your test.

## Tagging Strategy

| Tag | When to Use |
|---|---|
| `1.0.0` | Production release — always deploy with an exact version |
| `latest` | Development convenience — never deploy `latest` to production |
| `dev` | Local development builds |
| `abc1234` (git SHA) | CI builds — enables rollback to exact commit |
| `1.0.0-rc1` | Release candidates for staging |
