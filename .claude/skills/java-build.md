# Skill: java-build

## Overview

**Skill ID**: `java-build`
**Used by**: JavaBuildAgent
**Purpose**: Generates Maven build instructions for the LightCI Server Spring Boot application.

This skill defines the step-by-step procedure that JavaBuildAgent follows when
generating a build script. It covers prerequisite validation, Maven lifecycle execution,
artifact verification, and common error remediation.

---

## Prerequisites

| Requirement | Minimum Version | Verification Command |
|---|---|---|
| Java JDK | 17 (LTS) | `java -version` |
| Apache Maven | 3.6+ | `mvn -version` |
| `pom.xml` | (project root) | `test -f pom.xml` |
| Free disk space | 500 MB | `df -h .` |

## Step-by-Step Procedure

### Phase 1: Validation

```bash
# 1.1 — Verify Java version
java -version 2>&1 | grep -E 'version "17'
# Expected: openjdk version "17.x.x" ...

# 1.2 — Verify Maven is installed
mvn -version
# Expected: Apache Maven 3.x.x ...

# 1.3 — Verify pom.xml exists
test -f pom.xml && echo "pom.xml found" || { echo "ERROR: pom.xml missing"; exit 1; }

# 1.4 — Verify JAVA_HOME (optional but recommended)
echo "JAVA_HOME: ${JAVA_HOME:-not set}"
```

### Phase 2: Clean

```bash
# 2.1 — Remove previous build artifacts
mvn clean --batch-mode --no-transfer-progress
# This deletes: target/ directory and all its contents
```

### Phase 3: Compile

```bash
# 3.1 — Compile Java source files
mvn compile --batch-mode --no-transfer-progress
# Reads: src/main/java/**/*.java
# Writes: target/classes/**/*.class
```

### Phase 4: Test (optional)

```bash
# 4.1 — Run unit tests
mvn test --batch-mode --no-transfer-progress
# Reads: src/test/java/**/*.java
# Writes: target/test-classes/, target/surefire-reports/

# OR skip tests
mvn test -DskipTests --batch-mode --no-transfer-progress
```

### Phase 5: Package

```bash
# 5.1 — Create the executable fat JAR
mvn package -DskipTests --batch-mode --no-transfer-progress
# The spring-boot-maven-plugin repackages the JAR into a fat JAR
# Output: target/light-ci-server.jar
```

### Phase 6: Verify

```bash
# 6.1 — Confirm artifact exists
test -f target/light-ci-server.jar || { echo "ERROR: JAR missing"; exit 1; }

# 6.2 — Print size
ls -lh target/light-ci-server.jar

# 6.3 — Confirm it's a valid Spring Boot JAR
unzip -p target/light-ci-server.jar META-INF/MANIFEST.MF | grep Spring-Boot-Version
```

## Complete Generated Script

```bash
#!/bin/bash
# Generated by java-build skill — LightCI Server
set -euo pipefail

cd "$(git rev-parse --show-toplevel 2>/dev/null || pwd)"

echo "[1/4] Validating environment..."
java -version 2>&1 | grep -E 'version "17' || {
  echo "ERROR: Java 17 required"
  exit 1
}
mvn -version > /dev/null
test -f pom.xml

echo "[2/4] Cleaning..."
mvn clean --batch-mode --no-transfer-progress

echo "[3/4] Building..."
mvn package -DskipTests \
  --batch-mode \
  --no-transfer-progress \
  2>&1 | tee target/build.log

echo "[4/4] Verifying..."
JAR="target/light-ci-server.jar"
test -f "${JAR}" && echo "SUCCESS: $(du -sh ${JAR} | cut -f1) — ${JAR}" || exit 1
```

## How to Verify Success

```bash
# 1. JAR file exists
ls -lh target/light-ci-server.jar

# 2. JAR is executable (Spring Boot fat JAR)
java -jar target/light-ci-server.jar --spring.main.web-application-type=none --spring.profiles.active=test 2>&1 | head -5
# Should start and then stop (or error on missing DB, etc.) — but NOT "Unable to access jarfile"

# 3. Build log shows success
grep "BUILD SUCCESS" target/build.log

# 4. No test failures (if tests were run)
grep -E "Tests run:.*Failures: 0, Errors: 0" target/build.log
```

## Common Errors and Fixes

### `Could not resolve dependencies`
**Cause**: Network issue or missing repository configuration.
**Fix**:
```bash
mvn clean package -U -DskipTests  # -U forces dependency update
```

### `COMPILATION ERROR — cannot find symbol`
**Cause**: Missing import, typo, or a dependency not in `pom.xml`.
**Fix**: Read the specific error in `target/build.log` and fix the Java source or add the dependency.

### `OutOfMemoryError: Java heap space`
**Cause**: Maven JVM has insufficient memory.
**Fix**:
```bash
export MAVEN_OPTS="-Xmx2g -XX:MaxMetaspaceSize=512m"
mvn clean package -DskipTests
```

### `java.lang.UnsupportedClassVersionError`
**Cause**: Source compiled with Java 17 but Maven is using an older JDK.
**Fix**:
```bash
export JAVA_HOME=$(/usr/libexec/java_home -v 17)  # macOS
mvn clean package -DskipTests
```

### `Process forked JVM terminated abnormally`
**Cause**: Surefire (test runner) process crashed, often due to insufficient memory.
**Fix**:
```bash
mvn clean package -DskipTests  # Skip tests to confirm build itself works
# Then investigate test failures separately: mvn test -fae
```

### `JAR is 0 bytes or very small`
**Cause**: Build succeeded but the Spring Boot repackage plugin didn't run.
**Fix**: Confirm `spring-boot-maven-plugin` is in `pom.xml` `<build><plugins>` section.

## Maven Goal Reference

| Goal | What it does |
|---|---|
| `mvn clean` | Deletes `target/` |
| `mvn compile` | Compiles `src/main/java` |
| `mvn test` | Compiles + runs unit tests |
| `mvn package` | Compiles + tests + creates JAR |
| `mvn verify` | Package + integration tests |
| `mvn install` | Verify + installs to local `~/.m2` |
| `mvn clean package` | Clean then full package (recommended) |
| `mvn clean package -DskipTests` | Clean + package, skip tests |

---

## Notes

- Always use `mvn clean package`, not just `mvn package`. The `clean` phase ensures
  stale class files from previous builds don't contaminate the output.
- The `--batch-mode` flag disables interactive prompts and makes output CI-friendly.
- The `--no-transfer-progress` flag suppresses download progress bars (reduces log noise).
- `tee target/build.log` captures Maven output to a file while still printing to console.
