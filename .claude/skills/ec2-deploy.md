# Skill: ec2-deploy

## Overview

**Skill ID**: `ec2-deploy`
**Used by**: DeploymentAgent
**Purpose**: Generates complete SSH-based deployment scripts for the LightCI Server
on an AWS EC2 instance.

This skill defines the procedure DeploymentAgent follows when generating
`scripts/deploy.sh` and `scripts/docker-install.sh`. Claude generates these scripts
as text files — the developer reviews and runs them. Claude never executes SSH
commands or remotely accesses EC2 instances.

---

## Prerequisites

| Requirement | Description |
|---|---|
| EC2 instance running | Provisioned via Terraform (terraform-ec2 skill) |
| Docker installed on EC2 | Via Terraform user_data or docker-install.sh |
| Docker image pushed | To Docker Hub or ECR (docker-push skill) |
| SSH key available | `.pem` file with `chmod 400` |
| Port 22 open | Security group allows SSH from developer's IP |
| Port 8080 open | Security group allows app traffic |

---

## Inputs

| Input | Type | Default | Description |
|---|---|---|---|
| `EC2_PUBLIC_IP` | env var | required | Public IPv4 of the EC2 instance |
| `SSH_KEY_PATH` | env var | required | Local path to `.pem` file |
| `DOCKER_USERNAME` | env var | required | Docker Hub username |
| `DOCKER_IMAGE` | env var | `light-ci-server` | Docker image name |
| `DOCKER_TAG` | env var | `latest` | Docker image tag |
| `APP_PORT` | env var | `8080` | Container port to expose |
| `CONTAINER_NAME` | string | `light-ci-server` | Name for the Docker container |
| `SSH_USER` | string | `ec2-user` | SSH username (Amazon Linux default) |

## Step-by-Step Procedure

### Phase 1: Pre-Deployment Validation (local)

```bash
# 1.1 — Verify SSH key exists and has correct permissions
test -f "${SSH_KEY_PATH}" || { echo "ERROR: SSH key not found: ${SSH_KEY_PATH}"; exit 1; }
chmod 400 "${SSH_KEY_PATH}"

# 1.2 — Test SSH connectivity (does NOT deploy)
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    -o ConnectTimeout=10 \
    -o BatchMode=yes \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "echo 'SSH connection OK'"

# 1.3 — Verify Docker is installed on EC2
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "docker --version" || {
  echo "ERROR: Docker not found on EC2. Run: bash scripts/docker-install.sh"
  exit 1
}
```

### Phase 2: Registry Authentication on EC2

```bash
# 2.1 — Login to Docker Hub from EC2 (for private images)
# For public images, this step can be skipped
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "echo '${DOCKER_PASSWORD}' | docker login --username '${DOCKER_USERNAME}' --password-stdin"
```

### Phase 3: Pull Docker Image on EC2

```bash
# 3.1 — Pull the target image
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "docker pull '${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}'"
```

### Phase 4: Stop Old Container

```bash
# 4.1 — Gracefully stop and remove existing container
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "docker stop '${CONTAINER_NAME}' 2>/dev/null || true && docker rm '${CONTAINER_NAME}' 2>/dev/null || true"
```

### Phase 5: Start New Container

```bash
# 5.1 — Run the new container
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "docker run -d \
      --name '${CONTAINER_NAME}' \
      --restart unless-stopped \
      -p '${APP_PORT}:${APP_PORT}' \
      -e SPRING_PROFILES_ACTIVE=prod \
      '${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}'"
```

### Phase 6: Health Check

```bash
# 6.1 — Wait for Spring Boot to start (takes ~20-30 seconds)
sleep 30

# 6.2 — Check health from local machine
HTTP_STATUS=$(curl -s -o /dev/null -w "%{http_code}" \
  --max-time 15 \
  "http://${EC2_PUBLIC_IP}:${APP_PORT}/actuator/health")

echo "Health check: HTTP ${HTTP_STATUS}"

# 6.3 — Or check from inside EC2
ssh -i "${SSH_KEY_PATH}" -o StrictHostKeyChecking=no "${SSH_USER}@${EC2_PUBLIC_IP}" \
  "curl -s http://localhost:${APP_PORT}/actuator/health"
```

## Complete Generated Script: deploy.sh

```bash
#!/bin/bash
# Generated by ec2-deploy skill — LightCI Server Deployment
set -euo pipefail

EC2_PUBLIC_IP="${EC2_PUBLIC_IP:?Required}"
SSH_KEY_PATH="${SSH_KEY_PATH:?Required}"
DOCKER_USERNAME="${DOCKER_USERNAME:?Required}"
DOCKER_IMAGE="${DOCKER_IMAGE:-light-ci-server}"
DOCKER_TAG="${DOCKER_TAG:-latest}"
APP_PORT="${APP_PORT:-8080}"
CONTAINER_NAME="${CONTAINER_NAME:-light-ci-server}"
SSH_USER="${SSH_USER:-ec2-user}"
IMAGE_REF="${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"

echo "Deploying ${IMAGE_REF} to ${SSH_USER}@${EC2_PUBLIC_IP}:${APP_PORT}"

chmod 400 "${SSH_KEY_PATH}"
SSH="ssh -i ${SSH_KEY_PATH} -o StrictHostKeyChecking=no -o ConnectTimeout=30 ${SSH_USER}@${EC2_PUBLIC_IP}"

echo "[1/4] Pulling image..."
${SSH} "docker pull '${IMAGE_REF}'"

echo "[2/4] Stopping old container..."
${SSH} "docker stop '${CONTAINER_NAME}' 2>/dev/null || true && docker rm '${CONTAINER_NAME}' 2>/dev/null || true"

echo "[3/4] Starting new container..."
${SSH} "docker run -d --name '${CONTAINER_NAME}' --restart unless-stopped \
  -p '${APP_PORT}:${APP_PORT}' \
  -e SPRING_PROFILES_ACTIVE=prod \
  '${IMAGE_REF}'"

echo "[4/4] Health check (waiting 30s)..."
sleep 30
HTTP=$(curl -s -o /dev/null -w "%{http_code}" \
  --max-time 15 "http://${EC2_PUBLIC_IP}:${APP_PORT}/actuator/health" || echo 000)
echo "HTTP ${HTTP} — http://${EC2_PUBLIC_IP}:${APP_PORT}"
```

## Complete Generated Script: docker-install.sh

```bash
#!/bin/bash
# Generated by ec2-deploy skill — Docker Installation for EC2
set -euo pipefail

EC2_PUBLIC_IP="${EC2_PUBLIC_IP:?Required}"
SSH_KEY_PATH="${SSH_KEY_PATH:?Required}"
SSH_USER="${SSH_USER:-ec2-user}"

echo "Installing Docker on ${SSH_USER}@${EC2_PUBLIC_IP}..."

ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    "${SSH_USER}@${EC2_PUBLIC_IP}" << 'REMOTE'
set -e
dnf update -y
dnf install -y docker curl wget jq
systemctl enable docker
systemctl start docker
usermod -aG docker ec2-user
echo "Docker $(docker --version) installed."
REMOTE

echo "Done. Log back in to EC2 for group changes to take effect."
```

## How to Verify Success

```bash
# 1. Health check
curl http://${EC2_PUBLIC_IP}:8080/actuator/health
# Expected: {"status":"UP"}

# 2. Check container is running
ssh -i "${SSH_KEY_PATH}" ec2-user@${EC2_PUBLIC_IP} "docker ps"
# Expected: light-ci-server  Up X minutes  0.0.0.0:8080->8080/tcp

# 3. Check container logs
ssh -i "${SSH_KEY_PATH}" ec2-user@${EC2_PUBLIC_IP} "docker logs light-ci-server --tail 20"
# Expected: Started LightCiServerApplication in X.XXX seconds

# 4. Open app in browser
echo "http://${EC2_PUBLIC_IP}:8080"
```

## Common Errors and Fixes

### `Permission denied (publickey)`
**Cause**: Wrong key file or wrong permissions.
**Fix**: `chmod 400 "${SSH_KEY_PATH}"` and verify the key matches the key pair name in EC2.

### `Connection timed out`
**Cause**: EC2 not running, or security group blocks port 22.
**Fix**: Check EC2 state in AWS Console; verify SG ingress rule for port 22.

### `docker: command not found` (on EC2)
**Fix**: Run `bash scripts/docker-install.sh` first.

### `pull access denied` (private image)
**Fix**: Add Docker Hub login step before pulling. Ensure `DOCKER_PASSWORD` is set.

### Container starts but health check returns 000
**Cause**: Spring Boot is still starting (~30s needed).
**Fix**: Increase the `sleep` delay in deploy.sh from 30 to 60 seconds.

### `Bind for 0.0.0.0:8080 failed: port already allocated`
**Cause**: Old container wasn't properly stopped.
**Fix**: SSH to EC2 and run `docker ps -a` to find all containers, then `docker stop` and `docker rm` them manually.

## Rollback Procedure

```bash
# Rollback to a specific previous version
export DOCKER_TAG="1.0.0"  # the version to roll back to
bash scripts/deploy.sh
```

This stops the current container and starts the previous version in one operation.

---

## Notes

- The `--restart unless-stopped` flag ensures the container restarts automatically
  after EC2 reboots. This is important for production environments.
- Use `--memory="512m" --cpus="0.5"` in the docker run command to enforce resource
  limits on t3.micro (1 GB RAM) instances.
- Always use a specific image tag (not `latest`) in production for repeatable rollback.
- For zero-downtime deploys, consider running two containers on different ports
  behind a load balancer, or use AWS Elastic Beanstalk / ECS.
