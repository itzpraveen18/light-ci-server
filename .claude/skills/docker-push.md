# Skill: docker-push

## Overview

**Skill ID**: `docker-push`
**Used by**: DockerAgent
**Purpose**: Generates Docker image push instructions for Docker Hub or AWS ECR.

This skill covers authenticating to a container registry, pushing the image, and
verifying the push succeeded. It supports both Docker Hub and AWS Elastic Container
Registry (ECR) workflows.

---

## Prerequisites

| Requirement | Description |
|---|---|
| Docker Engine running | `docker info` must succeed |
| Image built locally | `docker images ${DOCKER_USERNAME}/${DOCKER_IMAGE}` must show the image |
| Registry credentials | Docker Hub: username + access token; ECR: AWS CLI configured |
| Internet access | Required to push to any registry |

## Step-by-Step Procedure: Docker Hub

### Phase 1: Authenticate

```bash
# 1.1 — Login using access token (NOT your Docker Hub login password)
# Create access tokens at: https://hub.docker.com/settings/security
echo "${DOCKER_PASSWORD}" | docker login \
  --username "${DOCKER_USERNAME}" \
  --password-stdin

# Expected: Login Succeeded
```

### Phase 2: Tag (if not already tagged)

```bash
# 2.1 — Ensure the versioned tag exists
docker images "${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}" | grep "${DOCKER_IMAGE}" || {
  echo "ERROR: Image ${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG} not found locally."
  echo "Run docker-build skill first."
  exit 1
}
```

### Phase 3: Push

```bash
# 3.1 — Push the versioned tag
docker push "${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"

# 3.2 — Push the latest tag (dev/staging only — never 'latest' in production)
docker push "${DOCKER_USERNAME}/${DOCKER_IMAGE}:latest"
```

### Phase 4: Verify

```bash
# 4.1 — Verify the image is accessible in Docker Hub
docker manifest inspect "${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"
# Should show: schemaVersion, mediaType, layers list

# 4.2 — Check in browser
echo "Verify at: https://hub.docker.com/r/${DOCKER_USERNAME}/${DOCKER_IMAGE}/tags"
```

## Step-by-Step Procedure: AWS ECR

### Phase 1: Create ECR Repository (first time only)

```bash
# 1.1 — Create the repository if it doesn't exist
aws ecr create-repository \
  --repository-name "${DOCKER_IMAGE}" \
  --region "${AWS_REGION}" \
  --image-scanning-configuration scanOnPush=true \
  --encryption-configuration encryptionType=AES256 \
  2>/dev/null || echo "Repository already exists"

# Set the ECR registry URL
ECR_REGISTRY="${AWS_ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com"
ECR_IMAGE="${ECR_REGISTRY}/${DOCKER_IMAGE}:${DOCKER_TAG}"
```

### Phase 2: Authenticate to ECR

```bash
# 2.1 — Get ECR token and login (token is valid for 12 hours)
aws ecr get-login-password \
  --region "${AWS_REGION}" | \
  docker login \
    --username AWS \
    --password-stdin \
    "${ECR_REGISTRY}"

# Expected: Login Succeeded
```

### Phase 3: Tag for ECR

```bash
# 3.1 — Tag the local image with the ECR reference
LOCAL_IMAGE="${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"
docker tag "${LOCAL_IMAGE}" "${ECR_IMAGE}"
echo "Tagged for ECR: ${ECR_IMAGE}"
```

### Phase 4: Push to ECR

```bash
# 4.1 — Push to ECR
docker push "${ECR_IMAGE}"

# Also push latest for ECR
docker tag "${LOCAL_IMAGE}" "${ECR_REGISTRY}/${DOCKER_IMAGE}:latest"
docker push "${ECR_REGISTRY}/${DOCKER_IMAGE}:latest"
```

### Phase 5: Verify ECR Push

```bash
# 5.1 — List images in ECR
aws ecr list-images \
  --repository-name "${DOCKER_IMAGE}" \
  --region "${AWS_REGION}" \
  --query 'imageIds[*].[imageTag,imageDigest]' \
  --output table

# 5.2 — Describe the image
aws ecr describe-images \
  --repository-name "${DOCKER_IMAGE}" \
  --image-ids imageTag="${DOCKER_TAG}" \
  --region "${AWS_REGION}"
```

## Complete Generated Script: Docker Hub

```bash
#!/bin/bash
# Generated by docker-push skill — Docker Hub
set -euo pipefail

DOCKER_USERNAME="${DOCKER_USERNAME:?Required}"
DOCKER_PASSWORD="${DOCKER_PASSWORD:?Required}"
DOCKER_IMAGE="${DOCKER_IMAGE:-light-ci-server}"
DOCKER_TAG="${DOCKER_TAG:-latest}"
IMAGE_REF="${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"

echo "Pushing: ${IMAGE_REF}"

# Authenticate
echo "${DOCKER_PASSWORD}" | docker login \
  --username "${DOCKER_USERNAME}" \
  --password-stdin

# Verify image exists locally
docker image inspect "${IMAGE_REF}" > /dev/null || {
  echo "ERROR: Image not found locally: ${IMAGE_REF}"
  exit 1
}

# Push
docker push "${IMAGE_REF}"
docker push "${DOCKER_USERNAME}/${DOCKER_IMAGE}:latest"

echo "Push complete: ${IMAGE_REF}"

# Verify
docker manifest inspect "${IMAGE_REF}" | head -5
```

## How to Verify Success

```bash
# Docker Hub
docker manifest inspect myusername/light-ci-server:1.0.0
# Shows: image manifest JSON with layer digests

# ECR
aws ecr describe-images \
  --repository-name light-ci-server \
  --image-ids imageTag=1.0.0 \
  --region us-east-1

# Can also verify by attempting a fresh pull (from a clean environment)
docker pull myusername/light-ci-server:1.0.0
```

## Common Errors and Fixes

### `unauthorized: incorrect username or password`
**Fix**: Use a Docker Hub access token (not your login password).
Create at: `https://hub.docker.com/settings/security`

### `denied: requested access to the resource is denied` (Docker Hub)
**Cause**: Repository is private and credentials are wrong, or repository doesn't exist.
**Fix**: Create the repository on Docker Hub first, or make it public.

### `no basic auth credentials` (ECR)
**Cause**: ECR authentication has expired (tokens last 12 hours).
**Fix**: Re-run the `aws ecr get-login-password | docker login` command.

### `An error occurred (RepositoryNotFoundException)` (ECR)
**Cause**: The ECR repository doesn't exist yet.
**Fix**: Create it first with `aws ecr create-repository --repository-name light-ci-server`.

### `Error response from daemon: Get "https://...": dial tcp: lookup ... no such host`
**Cause**: DNS or network failure.
**Fix**: Check internet connectivity. If behind a proxy, configure Docker proxy settings.

### `toomanyrequests: You have reached your pull rate limit` (Docker Hub)
**Cause**: Docker Hub rate limits unauthenticated pulls (100/6h).
**Fix**: Login to Docker Hub before pulling: `docker login`.

## Security Notes

- **Never use your Docker Hub login password** — create a dedicated access token
  with minimal scope (Write for CI, Read for deployment).
- **Docker Hub access tokens** can be scoped to specific repositories.
- **ECR authentication** uses short-lived tokens (12h) generated from IAM —
  more secure than long-lived credentials.
- **For production CI** (GitHub Actions, Jenkins, etc.), store credentials in
  CI secrets, not in environment files.
- **Rotate Docker Hub tokens** regularly — at least quarterly.
