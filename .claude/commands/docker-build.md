# Command: /docker-build

## Description

Triggers **DockerAgent** to generate a complete Docker build, tag, and push script
for the LightCI Server image. Claude produces a ready-to-run shell script. Claude
does not execute any Docker commands.

---

## Agent

**DockerAgent** (`.claude/agents/DockerAgent.md`)
**Skills**: `.skills/docker-build.md`, `.skills/docker-push.md`

---

## Prerequisites

Before invoking `/docker-build`, ensure:
- `mvn clean package` has been run — `target/light-ci-server.jar` must exist.
- `Dockerfile` exists at the project root.
- Environment variables are set: `DOCKER_USERNAME`, `DOCKER_IMAGE`, `DOCKER_TAG`.
- Docker Desktop (Mac/Windows) or Docker Engine (Linux) is running.

---

## What Claude Generates

When you invoke `/docker-build`, Claude will:

1. Check for `Dockerfile` at the project root. If missing, generate one from
   `.claude/templates/Dockerfile.template`.
2. Verify `target/light-ci-server.jar` is referenced correctly in the Dockerfile.
3. Generate a complete build + tag + push shell script.
4. Include login, push, and verification commands.

---

## Generated Script

```bash
#!/bin/bash
# LightCI Server — Docker Build, Tag & Push Script
# Generated by DockerAgent via /docker-build command
# Review this script, then run: bash docker-build.sh

set -euo pipefail

# ---- Configuration ----
DOCKER_USERNAME="${DOCKER_USERNAME:?Set DOCKER_USERNAME before running this script}"
DOCKER_IMAGE="${DOCKER_IMAGE:-light-ci-server}"
DOCKER_TAG="${DOCKER_TAG:-latest}"
APP_PORT="${APP_PORT:-8080}"
BUILD_PLATFORM="${BUILD_PLATFORM:-linux/amd64}"

IMAGE_REF="${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"
IMAGE_LATEST="${DOCKER_USERNAME}/${DOCKER_IMAGE}:latest"

echo "============================================"
echo " LightCI Server — Docker Build & Push"
echo " Image:    ${IMAGE_REF}"
echo " Platform: ${BUILD_PLATFORM}"
echo "============================================"

# ---- Step 1: Check Docker daemon ----
echo ""
echo "[1/7] Checking Docker daemon..."
docker info > /dev/null 2>&1 || {
  echo "ERROR: Docker daemon is not running."
  echo "Start Docker Desktop (Mac/Windows) or: sudo systemctl start docker (Linux)"
  exit 1
}
echo "Docker is running: $(docker --version)"

# ---- Step 2: Check Dockerfile ----
echo ""
echo "[2/7] Checking Dockerfile..."
if [ ! -f "Dockerfile" ]; then
  echo "ERROR: Dockerfile not found at project root."
  echo "Generate one: ask Claude 'Use DockerAgent to generate a Dockerfile'"
  exit 1
fi
echo "Dockerfile found."

# ---- Step 3: Check JAR artifact ----
echo ""
echo "[3/7] Checking JAR artifact..."
if [ ! -f "target/light-ci-server.jar" ]; then
  echo "ERROR: target/light-ci-server.jar not found."
  echo "Build the application first: bash .claude/hooks/pre-build.sh && mvn clean package"
  exit 1
fi
JAR_SIZE=$(du -sh target/light-ci-server.jar | cut -f1)
echo "JAR found: target/light-ci-server.jar (${JAR_SIZE})"

# ---- Step 4: Build Docker image ----
echo ""
echo "[4/7] Building Docker image..."
docker build \
  --platform "${BUILD_PLATFORM}" \
  --build-arg JAR_FILE=target/light-ci-server.jar \
  --build-arg APP_PORT="${APP_PORT}" \
  -t "${IMAGE_REF}" \
  -f Dockerfile \
  .

echo "Image built: ${IMAGE_REF}"

# ---- Step 5: Tag as latest ----
echo ""
echo "[5/7] Tagging as latest..."
docker tag "${IMAGE_REF}" "${IMAGE_LATEST}"
echo "Tagged: ${IMAGE_LATEST}"

# ---- Step 6: Login to Docker Hub ----
echo ""
echo "[6/7] Logging in to Docker Hub..."
echo "${DOCKER_PASSWORD:?Set DOCKER_PASSWORD before running this script}" | \
  docker login --username "${DOCKER_USERNAME}" --password-stdin
echo "Login successful."

# ---- Step 7: Push image ----
echo ""
echo "[7/7] Pushing image to Docker Hub..."
docker push "${IMAGE_REF}"
docker push "${IMAGE_LATEST}"

echo ""
echo "============================================"
echo " PUSH SUCCESS"
echo ""
echo " Pushed: ${IMAGE_REF}"
echo " Pushed: ${IMAGE_LATEST}"
echo "============================================"
echo ""
echo "Verify push:"
echo "  docker manifest inspect ${IMAGE_REF}"
echo ""
echo "Next steps:"
echo "  1. Generate Terraform infra: use /infra-plan in Claude"
echo "  2. Generate deploy script:   use /deploy in Claude"
```

## AWS ECR Variant

If pushing to AWS ECR instead of Docker Hub, replace steps 6-7 with:

```bash
# ECR Login (requires AWS CLI configured)
aws ecr get-login-password --region "${AWS_REGION}" | \
  docker login \
    --username AWS \
    --password-stdin \
    "${AWS_ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com"

# ECR image reference
ECR_IMAGE="${AWS_ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com/${DOCKER_IMAGE}:${DOCKER_TAG}"
docker tag "${IMAGE_REF}" "${ECR_IMAGE}"
docker push "${ECR_IMAGE}"
```

## Multi-Architecture Build

To build for both AMD64 and ARM64 (e.g., for M1/M2 Mac local testing and EC2):

```bash
# Requires: docker buildx create --use
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  --build-arg JAR_FILE=target/light-ci-server.jar \
  -t "${IMAGE_REF}" \
  --push \
  .
```

## Verifying the Image Locally

Before pushing, test the image locally:

```bash
# Run locally
docker run -d \
  --name lightci-test \
  -p 8080:8080 \
  "${IMAGE_REF}"

# Verify health
curl http://localhost:8080/actuator/health

# Check logs
docker logs lightci-test

# Clean up
docker stop lightci-test && docker rm lightci-test
```

## Environment Variables Required

| Variable | Description |
|---|---|
| `DOCKER_USERNAME` | Your Docker Hub username |
| `DOCKER_PASSWORD` | Docker Hub access token (not your login password) |
| `DOCKER_IMAGE` | Image name (default: `light-ci-server`) |
| `DOCKER_TAG` | Image tag — use semver in production (e.g. `1.0.0`) |
| `APP_PORT` | Exposed application port (default: `8080`) |
