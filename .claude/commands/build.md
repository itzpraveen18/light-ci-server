# Command: /build

## Description

Triggers **JavaBuildAgent** to generate a complete Maven build script for the
LightCI Server project. Claude produces a ready-to-run shell script that the
developer reviews and executes manually. Claude does not run Maven.

---

## Agent

**JavaBuildAgent** (`.claude/agents/JavaBuildAgent.md`)
**Skill**: `.skills/java-build.md`

---

## What Claude Generates

When you invoke `/build`, Claude will:

1. Read `pom.xml` to confirm artifact ID (`light-ci-server`), version, and Java version.
2. Read `src/main/resources/application.properties` to note any build-relevant settings.
3. Generate a complete build validation and execution script.
4. Confirm the expected output artifact path and print its size.

---

## Generated Script

```bash
#!/bin/bash
# LightCI Server — Maven Build Script
# Generated by JavaBuildAgent via /build command
# Review this script, then run: bash build.sh

set -euo pipefail

PROJECT_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
cd "${PROJECT_ROOT}"

echo "============================================"
echo " LightCI Server — Maven Build"
echo " Working directory: $(pwd)"
echo "============================================"

# ---- Step 1: Validate Java 17 ----
echo ""
echo "[1/6] Checking Java version..."
JAVA_VER=$(java -version 2>&1 | head -1 | sed 's/.*version "\([0-9]*\).*/\1/')
if [ "${JAVA_VER}" != "17" ]; then
  echo "ERROR: Java 17 required, found Java ${JAVA_VER}"
  echo "Fix (macOS):  export JAVA_HOME=\$(/usr/libexec/java_home -v 17)"
  echo "Fix (Linux):  export JAVA_HOME=/usr/lib/jvm/java-17-openjdk"
  exit 1
fi
echo "Java 17 confirmed: $(java -version 2>&1 | head -1)"

# ---- Step 2: Validate Maven ----
echo ""
echo "[2/6] Checking Maven..."
mvn -version || {
  echo "ERROR: Maven not found on PATH."
  echo "Install: https://maven.apache.org/download.cgi"
  exit 1
}

# ---- Step 3: Validate pom.xml ----
echo ""
echo "[3/6] Checking pom.xml..."
test -f pom.xml || {
  echo "ERROR: pom.xml not found in $(pwd)"
  echo "Run this script from the project root directory."
  exit 1
}
echo "pom.xml found."

# ---- Step 4: Clean ----
echo ""
echo "[4/6] Cleaning previous build..."
mvn clean --batch-mode --no-transfer-progress

# ---- Step 5: Build ----
echo ""
echo "[5/6] Building application (skipping tests)..."
mvn package -DskipTests \
  --batch-mode \
  --no-transfer-progress \
  2>&1 | tee target/build.log

# Check Maven exit code
if [ "${PIPESTATUS[0]}" -ne 0 ]; then
  echo ""
  echo "ERROR: Maven build failed. Check target/build.log for details."
  exit 1
fi

# ---- Step 6: Verify artifact ----
echo ""
echo "[6/6] Verifying build artifact..."
JAR="target/light-ci-server.jar"

if [ ! -f "${JAR}" ]; then
  echo "ERROR: Expected JAR not found: ${JAR}"
  echo "Check target/build.log for compilation errors."
  exit 1
fi

JAR_SIZE=$(du -sh "${JAR}" | cut -f1)
JAR_BYTES=$(wc -c < "${JAR}" | tr -d ' ')

if [ "${JAR_BYTES}" -lt 1000 ]; then
  echo "ERROR: JAR file appears empty or too small (${JAR_BYTES} bytes)"
  exit 1
fi

echo ""
echo "============================================"
echo " BUILD SUCCESS"
echo ""
echo " Artifact : ${JAR}"
echo " Size     : ${JAR_SIZE}"
echo " Java     : 17"
echo " Profile  : default"
echo "============================================"
echo ""
echo "Next steps:"
echo "  1. Run post-build validation: bash .claude/hooks/post-build.sh"
echo "  2. Build Docker image:        use /docker-build command in Claude"
```

---

## Optional: Build with Tests

To build including tests, modify step 5 to:

```bash
mvn package \
  --batch-mode \
  --no-transfer-progress \
  2>&1 | tee target/build.log
```

---

## Optional: Build with Maven Profile

To activate a specific Spring profile during build:

```bash
mvn package -DskipTests \
  -P production \
  --batch-mode \
  --no-transfer-progress
```

---

## Verifying the JAR

After building, verify the JAR is a valid Spring Boot fat JAR:

```bash
# Check manifest
unzip -p target/light-ci-server.jar META-INF/MANIFEST.MF

# Should contain:
# Main-Class: org.springframework.boot.loader.JarLauncher
# Start-Class: com.example.lightci.LightCiServerApplication

# Check JAR size (should be ~30-60 MB for a typical Spring Boot app)
ls -lh target/light-ci-server.jar

# Test run locally (optional):
java -jar target/light-ci-server.jar
# Then: curl http://localhost:8080/actuator/health
```

---

## Pre-build Validation

Before running the build script, run the pre-build hook to validate the environment:

```bash
bash .claude/hooks/pre-build.sh
```

The hook exits with code 1 if Java 17, Maven, or required env vars are missing.

---

## Post-build Validation

After a successful build, run:

```bash
bash .claude/hooks/post-build.sh
```

This confirms the JAR exists, is non-empty, and prints the artifact size.
