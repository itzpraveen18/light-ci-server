# Command: /deploy

## Description

Triggers **DeploymentAgent** to generate a complete EC2 deployment script for the
LightCI Server. Claude produces `scripts/deploy.sh` — a shell script the developer
runs manually. Claude never SSHes into EC2 or runs remote commands.

---

## Agent

**DeploymentAgent** (`.claude/agents/DeploymentAgent.md`)
**Skill**: `.skills/ec2-deploy.md`

---

## Prerequisites

Before invoking `/deploy`, ensure:
- Docker image has been pushed to Docker Hub: run `/docker-build` first.
- EC2 instance is running: run `/infra-plan`, then `terraform apply`.
- Environment variables are set: `EC2_PUBLIC_IP`, `SSH_KEY_PATH`, `DOCKER_USERNAME`,
  `DOCKER_IMAGE`, `DOCKER_TAG`.
- Docker is installed on the EC2 instance (Terraform `user_data` does this automatically).

---

## What Claude Generates

When you invoke `/deploy`, Claude will:

1. Read existing `scripts/deploy.sh` if present (to avoid overwriting customizations).
2. Generate a complete deployment script with SSH commands, Docker pull, and health check.
3. Generate `scripts/docker-install.sh` as a standalone Docker setup script.
4. Output the scripts as files — the developer reviews and executes them.

## Generated File: scripts/deploy.sh

```bash
#!/bin/bash
# LightCI Server — EC2 Deployment Script
# Generated by DeploymentAgent via /deploy command
#
# USAGE:
#   source .env          # Load environment variables
#   bash scripts/deploy.sh
#
# REQUIRED ENV VARS:
#   EC2_PUBLIC_IP    — Public IP of EC2 instance
#   SSH_KEY_PATH     — Path to .pem private key
#   DOCKER_USERNAME  — Docker Hub username
#   DOCKER_IMAGE     — Image name (default: light-ci-server)
#   DOCKER_TAG       — Image tag (default: latest)
#   APP_PORT         — Application port (default: 8080)
#
# DO NOT EXECUTE via Claude — review and run manually.

set -euo pipefail

# ---- Configuration ----
EC2_PUBLIC_IP="${EC2_PUBLIC_IP:?EC2_PUBLIC_IP is required}"
SSH_KEY_PATH="${SSH_KEY_PATH:?SSH_KEY_PATH is required}"
DOCKER_USERNAME="${DOCKER_USERNAME:?DOCKER_USERNAME is required}"
DOCKER_IMAGE="${DOCKER_IMAGE:-light-ci-server}"
DOCKER_TAG="${DOCKER_TAG:-latest}"
APP_PORT="${APP_PORT:-8080}"
CONTAINER_NAME="${CONTAINER_NAME:-light-ci-server}"
SSH_USER="${SSH_USER:-ec2-user}"
IMAGE_REF="${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"

echo "============================================"
echo " LightCI Server — EC2 Deployment"
echo ""
echo " Target:    ${SSH_USER}@${EC2_PUBLIC_IP}"
echo " Image:     ${IMAGE_REF}"
echo " Port:      ${APP_PORT}"
echo " Container: ${CONTAINER_NAME}"
echo "============================================"

# ---- Pre-flight checks ----
echo ""
echo "[PRE-FLIGHT] Running pre-deployment checks..."

if [ ! -f "${SSH_KEY_PATH}" ]; then
  echo "ERROR: SSH key not found: ${SSH_KEY_PATH}"
  exit 1
fi
chmod 400 "${SSH_KEY_PATH}"
echo "  SSH key:    OK (${SSH_KEY_PATH})"

# Test SSH connectivity (does not deploy anything)
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    -o ConnectTimeout=10 \
    -o BatchMode=yes \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "echo 'SSH connection OK'" || {
  echo "ERROR: Cannot SSH into ${EC2_PUBLIC_IP}"
  echo "Check: security group allows port 22, instance is running, key is correct."
  exit 1
}
echo "  SSH:        OK"

# Confirm Docker is installed on EC2
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    -o ConnectTimeout=10 \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "docker --version" > /dev/null 2>&1 || {
  echo "ERROR: Docker not found on EC2."
  echo "Run: bash scripts/docker-install.sh"
  exit 1
}
echo "  Docker:     OK (on EC2)"
echo ""

# ---- Deploy ----
echo "[1/4] Pulling image on EC2..."
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "docker pull ${IMAGE_REF}"

echo "[2/4] Stopping existing container..."
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "docker stop ${CONTAINER_NAME} 2>/dev/null || true && docker rm ${CONTAINER_NAME} 2>/dev/null || true"

echo "[3/4] Starting new container..."
ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    "${SSH_USER}@${EC2_PUBLIC_IP}" \
    "docker run -d \
      --name ${CONTAINER_NAME} \
      --restart unless-stopped \
      -p ${APP_PORT}:${APP_PORT} \
      -e SPRING_PROFILES_ACTIVE=prod \
      ${IMAGE_REF}"

echo "[4/4] Waiting for application startup (30s)..."
sleep 30

# ---- Health check ----
echo ""
echo "[HEALTH CHECK] Verifying application..."
HTTP_STATUS=$(curl -s -o /dev/null -w "%{http_code}" \
  --max-time 15 \
  "http://${EC2_PUBLIC_IP}:${APP_PORT}/actuator/health" || echo "000")

echo "  HTTP status: ${HTTP_STATUS}"

if [ "${HTTP_STATUS}" = "200" ]; then
  echo ""
  echo "============================================"
  echo " DEPLOYMENT SUCCESS"
  echo ""
  echo " App URL:    http://${EC2_PUBLIC_IP}:${APP_PORT}"
  echo " Health:     http://${EC2_PUBLIC_IP}:${APP_PORT}/actuator/health"
  echo " Container:  ${CONTAINER_NAME}"
  echo "============================================"
else
  echo ""
  echo "WARNING: Health check returned HTTP ${HTTP_STATUS}"
  echo "The container may still be starting. Check logs:"
  echo ""
  echo "  ssh -i ${SSH_KEY_PATH} ${SSH_USER}@${EC2_PUBLIC_IP} \\"
  echo "    'docker logs ${CONTAINER_NAME} --tail 50'"
fi
```

## Generated File: scripts/docker-install.sh

```bash
#!/bin/bash
# LightCI Server — Docker Installation Script for EC2
# Generated by DeploymentAgent via /deploy command
# Run once when provisioning a new EC2 instance
# (Not needed if Terraform user_data was applied successfully)
#
# USAGE: bash scripts/docker-install.sh
# DO NOT EXECUTE via Claude — review and run manually.

set -euo pipefail

EC2_PUBLIC_IP="${EC2_PUBLIC_IP:?EC2_PUBLIC_IP is required}"
SSH_KEY_PATH="${SSH_KEY_PATH:?SSH_KEY_PATH is required}"
SSH_USER="${SSH_USER:-ec2-user}"

echo "Installing Docker on EC2 at ${EC2_PUBLIC_IP}..."

ssh -i "${SSH_KEY_PATH}" \
    -o StrictHostKeyChecking=no \
    -o ConnectTimeout=30 \
    "${SSH_USER}@${EC2_PUBLIC_IP}" << 'REMOTE'

set -e
echo "[1/4] Updating packages (Amazon Linux 2023)..."
sudo dnf update -y

echo "[2/4] Installing Docker..."
sudo dnf install -y docker

echo "[3/4] Enabling Docker service..."
sudo systemctl enable docker
sudo systemctl start docker

echo "[4/4] Configuring user permissions..."
sudo usermod -aG docker ec2-user
sudo systemctl restart docker

docker --version
echo "Docker installation complete."

REMOTE

echo ""
echo "Docker installed on ${EC2_PUBLIC_IP}"
echo "Note: Log out and back in to EC2 for group membership to take effect."
echo "Verify: ssh -i ${SSH_KEY_PATH} ${SSH_USER}@${EC2_PUBLIC_IP} 'docker ps'"
```

## Post-Deploy Verification

After running `deploy.sh`, run the post-deploy hook to verify from your local machine:

```bash
bash .claude/hooks/post-deploy.sh
```

This curls the health endpoint and prints the application URL.

---

## Rollback Procedure

If the deployment fails, roll back to the previous image tag:

```bash
# Set the previous tag
export DOCKER_TAG="1.0.0"  # previous working version
bash scripts/deploy.sh
```

---

## Environment Variables Required

| Variable | Description |
|---|---|
| `EC2_PUBLIC_IP` | Public IP from `terraform output instance_public_ip` |
| `SSH_KEY_PATH` | Path to your `.pem` key (e.g. `~/keys/lightci.pem`) |
| `DOCKER_USERNAME` | Docker Hub username |
| `DOCKER_IMAGE` | Image name (default: `light-ci-server`) |
| `DOCKER_TAG` | Tag to deploy — pin to a specific version in production |
| `APP_PORT` | Application port (default: `8080`) |
