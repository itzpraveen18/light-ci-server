# JavaBuildAgent

## Purpose

JavaBuildAgent handles the Maven build lifecycle for the LightCI Server project.
It generates complete, ready-to-run Maven build scripts based on the project's
`pom.xml` configuration. It never executes Maven commands directly — it produces
shell scripts and instructions that the developer reviews and runs.

---

## Skills Used

- **Primary**: `.skills/java-build.md`

---

## Inputs

| Input | Type | Default | Description |
|---|---|---|---|
| `pom.xml` path | string | `./pom.xml` | Path to Maven project descriptor |
| Maven goals | string | `clean package` | Space-separated Maven lifecycle goals |
| Java version | integer | `17` | Required Java version |
| Skip tests | boolean | `false` | Whether to pass `-DskipTests` |
| Maven profile | string | (none) | Active Maven profile (e.g. `production`) |
| Maven opts | string | (none) | Extra JVM args for Maven itself |
| Threads | string | (none) | Parallel build threads (e.g. `4` or `1C`) |

---

## Outputs

| Output | Location | Description |
|---|---|---|
| Build script | stdout / `.claude/logs/artifacts/` | Shell script with all mvn commands |
| JAR artifact | `target/light-ci-server.jar` | Executable fat JAR |
| Build log reference | `target/build.log` | Where Maven output is redirected |
| Verification commands | stdout | Commands to confirm JAR size and integrity |

## Step-by-Step Plan

When invoked, JavaBuildAgent generates the following plan:

### Step 1 — Validate prerequisites
Generate validation commands (pre-build.sh logic):
```bash
java -version 2>&1 | grep "17"
mvn -version 2>&1 | grep "Apache Maven"
test -f pom.xml
```

### Step 2 — Clean previous build output
```bash
mvn clean
```

### Step 3 — Compile source code
```bash
mvn compile
```

### Step 4 — Run unit tests (unless skipped)
```bash
mvn test
# Or if skipping:
mvn test -DskipTests
```

### Step 5 — Package the application
```bash
mvn package -DskipTests
```

### Step 6 — Verify artifact exists
```bash
test -f target/light-ci-server.jar && echo "JAR found" || echo "ERROR: JAR missing"
ls -lh target/light-ci-server.jar
```

### Step 7 — (Optional) Run integration tests
```bash
mvn verify -DskipUnitTests
```

## Example Output Script

When `/build` is invoked, JavaBuildAgent produces a script similar to this:

```bash
#!/bin/bash
# Generated by JavaBuildAgent — LightCI Server Build Script
# Generated: $(date -u +"%Y-%m-%dT%H:%M:%SZ")
# DO NOT EXECUTE via Claude — review and run manually.

set -euo pipefail

PROJECT_ROOT="$(cd "$(dirname "$0")" && pwd)"
cd "$PROJECT_ROOT"

echo "============================================"
echo " LightCI Server — Maven Build"
echo "============================================"

# --- Prerequisite checks ---
echo "[1/5] Checking Java 17..."
java -version 2>&1 | grep -E "version \"17" || {
  echo "ERROR: Java 17 is required. Found:"
  java -version
  exit 1
}

echo "[2/5] Checking Maven..."
mvn -version || {
  echo "ERROR: Maven is not installed or not on PATH."
  exit 1
}

echo "[3/5] Verifying pom.xml..."
test -f pom.xml || {
  echo "ERROR: pom.xml not found in $(pwd)"
  exit 1
}

# --- Build ---
echo "[4/5] Running Maven build..."
mvn clean package -DskipTests \
  --batch-mode \
  --no-transfer-progress \
  2>&1 | tee target/build.log

# --- Verify ---
echo "[5/5] Verifying artifact..."
JAR="target/light-ci-server.jar"
if [ -f "$JAR" ]; then
  JAR_SIZE=$(du -sh "$JAR" | cut -f1)
  echo ""
  echo "============================================"
  echo " BUILD SUCCESS"
  echo " Artifact: $JAR"
  echo " Size:     $JAR_SIZE"
  echo "============================================"
else
  echo "ERROR: Expected artifact not found: $JAR"
  echo "Check target/build.log for details."
  exit 1
fi
```

## Error Scenarios and Handling

### Error: Java version mismatch
**Symptom**: `java -version` returns version other than 17.
**Generated fix**:
- On macOS with SDKMAN: `sdk use java 17.0.x-tem`
- On macOS with Homebrew: `export JAVA_HOME=$(/usr/libexec/java_home -v 17)`
- On Linux: `export JAVA_HOME=/usr/lib/jvm/java-17-openjdk`

### Error: pom.xml not found
**Symptom**: `test -f pom.xml` exits non-zero.
**Generated fix**: Ensure the script is run from the project root directory.

### Error: Maven compilation failure
**Symptom**: `[ERROR] COMPILATION ERROR` in Maven output.
**Generated response**: JavaBuildAgent will read the error lines from the log and
suggest specific fixes based on the compiler error messages.

### Error: Test failures
**Symptom**: `[ERROR] Tests run: X, Failures: Y` in Maven output.
**Generated options**:
1. Fix the failing tests (preferred).
2. Skip tests with `-DskipTests` (document the reason as a comment).

### Error: Insufficient memory
**Symptom**: `OutOfMemoryError` during compilation.
**Generated fix**:
```bash
export MAVEN_OPTS="-Xmx2g -XX:MaxMetaspaceSize=512m"
mvn clean package -DskipTests
```

### Error: Missing dependency
**Symptom**: `Could not resolve dependencies for project`.
**Generated fix**:
1. Check internet connectivity.
2. Try `mvn clean package -U` to force dependency updates.
3. Verify `~/.m2/settings.xml` for correct repository configuration.

### Error: Artifact already exists (stale)
**Symptom**: Old JAR with wrong timestamp in `target/`.
**Generated fix**: Always use `mvn clean package` — never `mvn package` alone.

## Notes

- JavaBuildAgent reads `pom.xml` to determine the correct artifact name and version.
- If a `Dockerfile` references a different JAR path than `target/light-ci-server.jar`,
  JavaBuildAgent will flag the discrepancy.
- JavaBuildAgent never modifies `pom.xml` unless explicitly instructed by the developer.
- Multi-module builds are not currently supported — this project is single-module.
