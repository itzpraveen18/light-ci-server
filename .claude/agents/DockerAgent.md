# DockerAgent

## Purpose

DockerAgent handles the complete Docker workflow for the LightCI Server:
image build, tagging, and push to a container registry. It generates
ready-to-run shell scripts and can generate a Dockerfile from the project
template if one is missing. It never executes Docker commands directly.

---

## Skills Used

- **Primary**: `.skills/docker-build.md`
- **Secondary**: `.skills/docker-push.md`

---

## Inputs

| Input | Type | Default | Description |
|---|---|---|---|
| `DOCKER_USERNAME` | env var | required | Docker Hub username or ECR account |
| `DOCKER_IMAGE` | env var | `light-ci-server` | Image repository name |
| `DOCKER_TAG` | env var | `latest` | Image tag (use semver in production) |
| Registry type | string | `dockerhub` | `dockerhub` or `ecr` |
| Dockerfile path | string | `./Dockerfile` | Path to Dockerfile |
| Build platform | string | `linux/amd64` | Target platform for `--platform` flag |
| Build args | map | (none) | Docker build-time `--build-arg` values |
| `AWS_REGION` | env var | (ECR only) | Required when registry is ECR |
| `AWS_ACCOUNT_ID` | env var | (ECR only) | Required when registry is ECR |

---

## Outputs

| Output | Location | Description |
|---|---|---|
| Build + push script | stdout | Shell script with all docker commands |
| Tagged image reference | stdout | Full registry/image:tag string |
| Dockerfile (if missing) | `./Dockerfile` | Generated from `.claude/templates/Dockerfile.template` |
| Push verification | stdout | `docker manifest inspect` command |

## Step-by-Step Plan

### Step 1 — Validate prerequisites
```bash
docker info > /dev/null 2>&1 || { echo "ERROR: Docker daemon not running"; exit 1; }
test -f Dockerfile || echo "WARNING: Dockerfile not found — will generate one"
test -f target/light-ci-server.jar || { echo "ERROR: JAR not built yet. Run /build first."; exit 1; }
```

### Step 2 — Determine image reference
For Docker Hub:
```
IMAGE_REF="${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"
```
For AWS ECR:
```
IMAGE_REF="${AWS_ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com/${DOCKER_IMAGE}:${DOCKER_TAG}"
```

### Step 3 — Build the image
```bash
docker build \
  --platform linux/amd64 \
  --build-arg JAR_FILE=target/light-ci-server.jar \
  --build-arg APP_PORT=8080 \
  -t "${IMAGE_REF}" \
  -f Dockerfile \
  .
```

### Step 4 — Tag with additional tags
```bash
# Also tag as latest for convenience
docker tag "${IMAGE_REF}" "${DOCKER_USERNAME}/${DOCKER_IMAGE}:latest"
```

### Step 5 — Authenticate to registry
For Docker Hub:
```bash
echo "${DOCKER_PASSWORD}" | docker login --username "${DOCKER_USERNAME}" --password-stdin
```
For ECR:
```bash
aws ecr get-login-password --region "${AWS_REGION}" | \
  docker login --username AWS --password-stdin \
  "${AWS_ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com"
```

### Step 6 — Push image
```bash
docker push "${IMAGE_REF}"
docker push "${DOCKER_USERNAME}/${DOCKER_IMAGE}:latest"
```

### Step 7 — Verify push
```bash
docker manifest inspect "${IMAGE_REF}"
echo "Image pushed successfully: ${IMAGE_REF}"
```

## Example Output Script (Docker Hub)

```bash
#!/bin/bash
# Generated by DockerAgent — LightCI Server Docker Build & Push
# Generated: $(date -u +"%Y-%m-%dT%H:%M:%SZ")
# DO NOT EXECUTE via Claude — review and run manually.

set -euo pipefail

# --- Configuration ---
DOCKER_USERNAME="${DOCKER_USERNAME:?DOCKER_USERNAME is required}"
DOCKER_IMAGE="${DOCKER_IMAGE:-light-ci-server}"
DOCKER_TAG="${DOCKER_TAG:-latest}"
APP_PORT="${APP_PORT:-8080}"
IMAGE_REF="${DOCKER_USERNAME}/${DOCKER_IMAGE}:${DOCKER_TAG}"

echo "============================================"
echo " LightCI Server — Docker Build & Push"
echo " Image: ${IMAGE_REF}"
echo "============================================"

# --- Check Docker daemon ---
echo "[1/6] Checking Docker..."
docker info > /dev/null 2>&1 || {
  echo "ERROR: Docker daemon is not running."
  exit 1
}

# --- Check JAR exists ---
echo "[2/6] Checking JAR artifact..."
test -f target/light-ci-server.jar || {
  echo "ERROR: target/light-ci-server.jar not found."
  echo "Run the build step first: bash .claude/hooks/pre-build.sh && mvn clean package"
  exit 1
}

# --- Build image ---
echo "[3/6] Building Docker image..."
docker build \
  --platform linux/amd64 \
  --build-arg JAR_FILE=target/light-ci-server.jar \
  --build-arg APP_PORT="${APP_PORT}" \
  -t "${IMAGE_REF}" \
  -f Dockerfile \
  .

# --- Tag as latest ---
echo "[4/6] Tagging as latest..."
docker tag "${IMAGE_REF}" "${DOCKER_USERNAME}/${DOCKER_IMAGE}:latest"

# --- Login ---
echo "[5/6] Logging in to Docker Hub..."
echo "${DOCKER_PASSWORD:?DOCKER_PASSWORD is required}" | \
  docker login --username "${DOCKER_USERNAME}" --password-stdin

# --- Push ---
echo "[6/6] Pushing image..."
docker push "${IMAGE_REF}"
docker push "${DOCKER_USERNAME}/${DOCKER_IMAGE}:latest"

echo ""
echo "============================================"
echo " PUSH SUCCESS"
echo " ${IMAGE_REF}"
echo " ${DOCKER_USERNAME}/${DOCKER_IMAGE}:latest"
echo "============================================"
```

## Generated Dockerfile (if missing)

If no `Dockerfile` is found at the project root, DockerAgent generates one
based on `.claude/templates/Dockerfile.template`:

```dockerfile
# Generated by DockerAgent — LightCI Server
# Multi-stage build: builder + runtime

# --- Stage 1: Builder ---
FROM eclipse-temurin:17-jdk-alpine AS builder
WORKDIR /build
COPY pom.xml .
COPY src ./src
RUN apk add --no-cache maven && \
    mvn dependency:go-offline -q && \
    mvn clean package -DskipTests -q

# --- Stage 2: Runtime ---
FROM eclipse-temurin:17-jre-alpine AS runtime
WORKDIR /app

RUN addgroup -S appgroup && adduser -S appuser -G appgroup

COPY --from=builder /build/target/light-ci-server.jar app.jar

RUN chown appuser:appgroup app.jar

USER appuser

EXPOSE 8080

ENV JAVA_OPTS="-XX:+UseContainerSupport -XX:MaxRAMPercentage=75.0 -Djava.security.egd=file:/dev/./urandom"

HEALTHCHECK --interval=30s --timeout=10s --start-period=60s --retries=3 \
  CMD wget -qO- http://localhost:8080/actuator/health || exit 1

ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -jar app.jar"]
```

## Error Scenarios and Handling

### Error: Docker daemon not running
**Symptom**: `Cannot connect to the Docker daemon`.
**Generated fix**: `sudo systemctl start docker` (Linux) or start Docker Desktop (Mac/Windows).

### Error: JAR file not found
**Symptom**: `COPY target/light-ci-server.jar` fails during build.
**Generated fix**: Run the Maven build step first. The JAR must exist before docker build.

### Error: Authentication failure
**Symptom**: `unauthorized: incorrect username or password`.
**Generated fix**: Verify `DOCKER_USERNAME` and `DOCKER_PASSWORD` env vars. Use a Docker Hub
access token (not your password) for CI environments.

### Error: Platform mismatch
**Symptom**: Image runs on local machine but fails on EC2 (ARM vs AMD64).
**Generated fix**: Add `--platform linux/amd64` to the docker build command. For multi-arch:
```bash
docker buildx build --platform linux/amd64,linux/arm64 -t "${IMAGE_REF}" --push .
```

### Error: No space left on device
**Symptom**: Docker build fails partway through with disk error.
**Generated fix**: `docker system prune -f` to remove unused images and layers.

### Error: ECR repository does not exist
**Symptom**: `name unknown: The repository with name ... does not exist`.
**Generated fix**: Create the ECR repository first:
```bash
aws ecr create-repository \
  --repository-name light-ci-server \
  --region "${AWS_REGION}"
```
